# 📰 News Digest Bot

Телеграм-бот, который раз в сутки собирает свежие новости из RSS-лент по вашим темам
и присылает дайджест в виде заголовков со ссылками.


## Как это работает

1. GitHub Actions по расписанию (cron) раз в сутки запускает `news_bot.py`.
2. Скрипт читает `config.yaml`, скачивает RSS-ленты, отбирает новости за последние
   24 часа и фильтрует их по ключевым словам.
3. Группирует по темам и отправляет в ваш Telegram через бесплатный Bot API.

## Настройка (15 минут, один раз)

### Шаг 1. Создайте бота и получите токен

1. В Telegram напишите [@BotFather](https://t.me/BotFather).
2. Команда `/newbot` → задайте имя и username.
3. BotFather пришлёт **токен** вида `123456789:AAE...`. Сохраните его.

### Шаг 2. Узнайте свой chat_id

1. Напишите своему боту любое сообщение (например, «привет»).
2. Откройте в браузере (подставив свой токен):
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates`
3. Найдите в ответе `"chat":{"id":123456789` — это число и есть ваш **chat_id**.

### Шаг 3. Залейте проект на GitHub

```bash
cd news-digest-bot
git init
git add .
git commit -m "News digest bot"
# создайте пустой репозиторий на github.com, затем:
git remote add origin https://github.com/ВАШ_ЛОГИН/news-digest-bot.git
git push -u origin main
```

> Репозиторий можно сделать **публичным** — тогда GitHub Actions полностью бесплатен
> без лимитов. Токен и chat_id в коде не хранятся (см. шаг 4), так что это безопасно.

### Шаг 4. Добавьте секреты

В репозитории: **Settings → Secrets and variables → Actions → New repository secret**.
Создайте два секрета:

| Имя                  | Значение                |
|----------------------|-------------------------|
| `TELEGRAM_BOT_TOKEN` | токен из шага 1         |
| `TELEGRAM_CHAT_ID`   | chat_id из шага 2       |

### Шаг 5. Настройте темы

Откройте `config.yaml` и впишите свои темы, ленты и ключевые слова
(примеры уже внутри). Закоммитьте и запушьте изменения.

### Шаг 6. Проверьте

Вкладка **Actions → Daily News Digest → Run workflow** — запуск вручную.
Через минуту в Telegram должен прийти дайджест.

## Время отправки

Cron в GitHub Actions работает **по UTC**. В `.github/workflows/daily.yml` стоит
`0 7 * * *` = **07:00 UTC = 10:00 МСК**. Поменяйте час под свой часовой пояс.

> ⚠️ GitHub иногда запускает запланированные задачи с задержкой 5–15 минут при
> высокой нагрузке — это нормально для бесплатного плана.

## Локальный запуск (для теста)

```powershell
pip install -r requirements.txt
$env:TELEGRAM_BOT_TOKEN = "ваш_токен"
$env:TELEGRAM_CHAT_ID   = "ваш_chat_id"
python news_bot.py
```

## Где брать RSS-ленты

- У большинства СМИ есть страница `сайт.com/rss`.
- **Google News по любому запросу** (удобно для произвольных тем):
  ```
  https://news.google.com/rss/search?q=ВАШ+ЗАПРОС&hl=ru&gl=RU&ceid=RU:ru
  ```
  Пробелы → `+`, точную фразу оборачивайте в `%22...%22`.
